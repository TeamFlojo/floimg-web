The floimg plugin provides six slash commands for different image generation tasks. Each command is optimized for its specific use case.

## /floimg-claude:image

**Universal image generation** - Generates any type of image based on your description.

### Usage

```
/floimg-claude:image <description>
```

### How It Works

1. Analyzes your description to determine the best generator
2. Routes to the appropriate backend (OpenAI, QuickChart, Mermaid, QR, Screenshot)
3. Generates the image and saves it locally

### Routing Keywords

| Keywords | Generator | Result |
|----------|-----------|--------|
| photo, picture, illustration, scene | OpenAI/DALL-E | AI-generated image |
| chart, graph, bar, line, pie | QuickChart | Data visualization |
| flowchart, diagram, sequence | Mermaid | Technical diagram |
| QR, qr code | QR | QR code |
| screenshot, capture, webpage | Screenshot | Webpage capture |

### Examples

```
/floimg-claude:image a golden retriever in a field at sunset
```

```
/floimg-claude:image bar chart of monthly sales data
```

```
/floimg-claude:image flowchart showing user registration process
```

## /floimg-claude:chart

**Data visualization** - Creates charts and graphs using QuickChart.

### Usage

```
/floimg-claude:chart <description with data>
```

### Supported Chart Types

| Type | Best For |
|------|----------|
| bar / horizontalBar | Categorical comparisons |
| line | Trends over time |
| pie / doughnut | Parts of a whole |
| radar | Multi-variable comparison |
| scatter / bubble | Correlation analysis |
| polarArea | Radial data display |

### Examples

**Simple bar chart:**
```
/floimg-claude:chart bar chart showing Q1: $100K, Q2: $150K, Q3: $175K, Q4: $200K
```

**Pie chart:**
```
/floimg-claude:chart pie chart of market share: Us 45%, Competitor A 30%, Others 25%
```

**Line chart with multiple series:**
```
/floimg-claude:chart line chart comparing 2023 vs 2024 revenue by month
```

### Under the Hood

Claude parses your description and structures it as Chart.js configuration:

```json
{
  "type": "bar",
  "data": {
    "labels": ["Q1", "Q2", "Q3", "Q4"],
    "datasets": [{
      "label": "Revenue",
      "data": [100, 150, 175, 200],
      "backgroundColor": ["#4ade80", "#22d3ee", "#a78bfa", "#f472b6"]
    }]
  },
  "options": {
    "plugins": {
      "title": { "display": true, "text": "Quarterly Revenue" }
    }
  }
}
```

## /floimg-claude:diagram

**Technical diagrams** - Generates Mermaid diagrams for documentation and architecture.

### Usage

```
/floimg-claude:diagram <description>
```

### Diagram Types

| Type | Syntax | Use Case |
|------|--------|----------|
| Flowchart | `graph TD` / `graph LR` | Process flows |
| Sequence | `sequenceDiagram` | API interactions |
| Class | `classDiagram` | OOP relationships |
| State | `stateDiagram-v2` | Lifecycles |
| Gantt | `gantt` | Project timelines |
| ER | `erDiagram` | Database schemas |
| Mindmap | `mindmap` | Brainstorming |

### Examples

**Process flow:**
```
/floimg-claude:diagram user login flow: enter credentials -> validate -> success or show error
```

**Sequence diagram:**
```
/floimg-claude:diagram sequence diagram of API auth: user -> API -> database -> response
```

**Architecture:**
```
/floimg-claude:diagram microservices architecture with frontend, API gateway, and three services
```

### Mermaid Syntax Reference

Claude generates valid Mermaid code. For flowcharts:

| Node Shape | Syntax |
|------------|--------|
| Rectangle | `A[Text]` |
| Diamond (decision) | `B{Text}` |
| Circle | `C((Text))` |
| Stadium | `D([Text])` |
| Database | `E[(Text)]` |

| Edge Type | Syntax |
|-----------|--------|
| Arrow | `-->` |
| Line | `---` |
| Dotted arrow | `-.->` |
| Thick arrow | `==>` |
| Labeled arrow | `--text-->` |

## /floimg-claude:qr

**QR code generation** - Creates QR codes for URLs, text, or structured data.

### Usage

```
/floimg-claude:qr <content>
```

### Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| text | Content to encode | (from command) |
| width | Size in pixels | 300 |
| errorCorrectionLevel | L, M, Q, or H | M |
| margin | Quiet zone modules | 4 |

### Error Correction Levels

| Level | Recovery | When to Use |
|-------|----------|-------------|
| L | 7% | Clean environments, max data |
| M | 15% | General use (recommended) |
| Q | 25% | Some expected damage |
| H | 30% | Logo overlay needed |

### Examples

**Simple URL:**
```
/floimg-claude:qr https://floimg.com
```

**With options:**
```
/floimg-claude:qr https://floimg.com/docs high quality, 500px
```

### Content Types

| Type | Format |
|------|--------|
| URL | `https://example.com` |
| Plain text | Any text string |
| WiFi | `WIFI:T:WPA;S:NetworkName;P:Password;;` |
| vCard | `BEGIN:VCARD...END:VCARD` |
| Email | `mailto:user@example.com` |
| Phone | `tel:+1234567890` |
| SMS | `sms:+1234567890?body=Hello` |

## /floimg-claude:screenshot

**Webpage capture** - Takes screenshots of any webpage using Playwright.

### Usage

```
/floimg-claude:screenshot <url> [options]
```

### Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| url | Webpage URL (required) | - |
| fullPage | Capture entire scrollable page | false |
| width | Viewport width in pixels | 1280 |
| height | Viewport height in pixels | 720 |
| deviceScaleFactor | Pixel density (2 for Retina) | 1 |

### Examples

**Basic screenshot:**
```
/floimg-claude:screenshot https://github.com
```

**Full page capture:**
```
/floimg-claude:screenshot https://docs.floimg.com full page
```

**Mobile viewport:**
```
/floimg-claude:screenshot https://example.com mobile view iPhone 14
```

### Common Viewport Sizes

| Device | Width | Height | Scale |
|--------|-------|--------|-------|
| Desktop HD | 1920 | 1080 | 1 |
| Desktop | 1280 | 720 | 1 |
| Laptop | 1366 | 768 | 1 |
| iPad Pro | 1024 | 1366 | 2 |
| iPad | 768 | 1024 | 2 |
| iPhone 14 Pro | 393 | 852 | 3 |
| iPhone SE | 375 | 667 | 2 |

### Notes

- Requires `@teamflojo/floimg-screenshot` plugin
- First screenshot may be slower (browser launch)
- JavaScript executes before capture
- Some sites may block automated screenshots

## /floimg-claude:workflow

**Multi-step pipelines** - Executes generate, transform, and save operations in sequence.

### Usage

```
/floimg-claude:workflow <description>
```

### Pipeline Structure

Workflows combine three operation types:

1. **Generate** - Create the initial image
2. **Transform** - Modify the image (resize, blur, caption, etc.)
3. **Save** - Store to filesystem or cloud

### Transform Operations

| Operation | Description | Key Parameters |
|-----------|-------------|----------------|
| resize | Scale image | width, height, fit |
| convert | Change format | to (png, jpeg, webp) |
| blur | Gaussian blur | sigma (0.3-1000) |
| sharpen | Sharpen edges | sigma |
| grayscale | Remove color | - |
| modulate | Adjust colors | brightness, saturation, hue |
| roundCorners | Border radius | radius |
| addText | Overlay text | text, x, y, size, color |
| addCaption | Caption bar | text, position, background |
| preset | Apply filter | name (vintage, vibrant, dramatic, soft) |

### Examples

**Social media image:**
```
/floimg-claude:workflow Create a hero image for my blog, resize to 1200x630, add "Welcome" caption, save to ./social/
```

**Thumbnail generation:**
```
/floimg-claude:workflow Generate a product photo, create 100px and 200px thumbnails with rounded corners
```

**Cloud upload:**
```
/floimg-claude:workflow Create a chart of Q1 sales, add company watermark, upload to S3 bucket
```

### Save Destinations

| Type | Format | Example |
|------|--------|---------|
| Local | Relative/absolute path | `./output/image.png` |
| S3 | s3://bucket/key | `s3://my-bucket/images/hero.png` |
| R2 | r2://bucket/key | `r2://cdn-bucket/assets/logo.png` |
| Tigris | tigris://bucket/key | `tigris://media/uploads/photo.jpg` |

### Best Practices

- **Order matters**: Resize before adding text/caption for quality
- **Use pipelines for multi-step**: More efficient than separate commands
- **Save to cloud last**: Apply all transforms before uploading
